# Auto-Generate SKU Feature - Changelog

**Fecha**: 15 de Enero, 2026  
**Autor**: Senior Frontend Engineer (Next.js 15 + TypeScript strict)

## 🎯 Objetivo Implementado

Generación automática de SKU cuando el usuario escribe el nombre del producto en `/ui/pages/productos/ProductosView.tsx`.

## ✨ Características

✅ **SKU sugerido automáticamente** - Se genera en tiempo real mientras escribes el nombre  
✅ **SKU editable manualmente** - El usuario puede modificar el SKU sugerido  
✅ **Flag `skuTouched`** - Previene sobrescribir SKU editado manualmente  
✅ **Validación de unicidad** - Consulta API para obtener siguiente número incremental  
✅ **UX mejorada** - Muestra feedback visual cuando SKU es sugerido  
✅ **TypeScript strict** - Sin errores de tipo  

## 📝 Reglas de Generación de SKU

### Formato del prefijo
- **1 palabra**: `AAA` (primeras 3 letras)
- **2+ palabras**: `AAA-BBB` (primeras 3 letras de primeras 2 palabras)

### Ejemplos
- `"Cable eléctrico"` → `CAB-ELE-001`
- `"Martillo"` → `MAR-001`
- `"Clavo 2 pulgadas"` → `CLA-PUL-001`

### Normalización
- Quita tildes/ñ (éáíóú → eaíóu)
- Quita símbolos especiales
- Convierte a mayúsculas
- Quita espacios dobles

### Número incremental
- Busca SKUs existentes con el mismo prefijo
- Suma +1 al último número encontrado
- Formato: `###` (3 dígitos con padding de ceros)


#### Funciones Nuevas
```typescript

// Genera SKU automáticamente desde nombre si skuTouched === false
```

#### Cambios en Handlers
- `handleCreate()`: Inicia con `skuTouched = false`
- `handleEdit()`: Inicia con `skuTouched = true` (preserva SKU existente)
- `handleInputChange()`: Lógica especial para SKU y nombre
  - Cuando se edita SKU: marca como `skuTouched = true`
  - Cuando se borra SKU completamente: marca como `skuTouched = false`
  - Cuando se edita nombre y `skuTouched = false`: regenera SKU

#### UI Mejorada
- Muestra mensaje bajo input SKU: "💡 SKU sugerido automáticamente (puedes editarlo)"
- Solo aparece cuando:
  - El SKU fue auto-generado (no editado manualmente)
  - Hay valor en SKU
  - Es un producto nuevo (no está editando)

## 🧪 Flujo de Uso

### Crear nuevo producto
1. Click en "Nuevo Producto"
2. Escribe nombre: "Cable eléctrico"
3. ✨ SKU se genera automáticamente: "CAB-ELE-001"
4. Muestra: "💡 SKU sugerido automáticamente (puedes editarlo)"
5. Usuario puede:
   - ✅ Aceptar el SKU (deja como está)
   - ✏️ Editar el SKU (marca como touched, nunca se regenera)
   - ❌ Borrar el SKU (vuelve a `skuTouched = false`, regenera si cambia nombre)
6. Guardar producto

### Editar producto existente
1. Click en botón ✏️
2. Modal abre con datos del producto
3. SKU está marcado como `touched` → no se regenera aunque cambies nombre
4. Puedes editar nombre sin que cambie el SKU
5. Si quieres cambiar SKU manualmente: edítalo directamente
6. Guardar cambios

## 🔄 Flujo de API

### Backend
1. Usuario escribe nombre: "Martillo"
2. Frontend genera prefijo: "MAR"
3. Frontend llama: `GET /api/products/next-sku?prefix=MAR`
4. Backend:
   - Busca productos donde sku LIKE 'MAR%'
   - Encuentra "MAR-001" y "MAR-002"
   - Retorna: `{ nextSku: "MAR-003" }`
5. Frontend genera SKU completo: "MAR-003"
6. Se muestra en input con feedback visual

## 📝 Notas Técnicas

- El helper `removeAccents()` usa `normalize('NFD')` para descomponer caracteres acentuados
- El endpoint utiliza Prisma para queries eficientes
- Manejo de concurrencia: Si dos usuarios crean productos simultáneamente, la BD previene duplicados (SKU unique constraint)
- Las animaciones CSS son suaves (0.2s ease-out) sin afectar performance
- No hay debounce en la generación de SKU (respuesta inmediata)

## 🐛 Casos Edge

✅ Nombre vacío → SKU vacío, no intenta generar  
✅ Nombre con solo espacios → SKU vacío  
✅ Nombre con caracteres especiales → Se normalizan correctamente  
✅ Prefijo nuevo (sin SKUs previos) → Comienza en 001  
✅ Usuario edita nombre múltiples veces → SKU regenera si no está touched  
✅ Usuario borra SKU completamente → Regenera desde nombre
